    Configure the firefox to hide TAB bar, address bar adn bookmark bar

    Enable CSS support

    1. Type “about: config” in the Firefox address bar and then press the Enter key
    2. Accept the risk and move on
    3. Search toolkit.legacyUserProfileCustomizations.stylesheets
    4. Make sure its value is set to true.

    Find and create the CSS file

    1. Type “about: support” in the Firefox and press Enter.
    2. Find the line “Profile Folder” and open it
    3. Check if there is a folder named chrome. if not ,create it.
    4. Create a file userChrome.css including the following code.
    /* --- 默认隐藏 标签栏、地址栏、书签栏 --- */
#TabsToolbar,
#nav-bar,
#PersonalToolbar {
  /* 使用 max-height 来实现平滑的“滑出”效果，而不是突然消失 */
  max-height: 0px !important;
  opacity: 0 !important;
  overflow: hidden !important;
  
  /* 动画效果 */而不是突然消失 */
  max-height: 0px !important;
  opacity: 0 !important;
  overflow: hidden !important;
  
  /* 动画效果 */
  transition: max-height 0.15s ease-out, opacity 0.15s ease-out !important;
}

    Explanation

    /* --- 默认隐藏 标签栏、地址栏、书签栏 --- */
#TabsToolbar,
#nav-bar,
#PersonalToolbar {
  /* ... */
}
    • These three lines are “selectors”. It tells Firefox: “select these three elements: #TabsToolbar (TAB bar), #nav-bar (address bar), #PersonalToolbar (bookmark bar)”
    • This selector has no conditions (such as :hover), so it defines what these toolbars look like by default.
    /* 使用 max-height 来实现平滑的“滑出”效果，而不是突然消失 */
  max-height: 0px !important;
    • Force (!important) the “maximum height” of these toolbars to 0 pixel.
    • This is the core to achieving the “folding” effect. It tells the browser that this element can only be up to 0 pixels high at most, thereby “flattening (压扁)” it to disappear in the vertical direction.
    opacity: 0 !important
    • Force the “Opacity (不透明度)” of the toolbar to be set to 0 (completely transparent 完全透明).
    • Make it completely invisible. This works in conjunction with max-height:0 to ensure that elements fade out while folding.
    overflow: hidden !important;
    • Forc the “overflowing content(溢出内容)” to be “hidden”
    • Even if the height of an element is 0, the content inside it (such as labels and buttons) is actually still there. The function of this line of code is to cut all content that exceeds the height of 0 pixels, so that they are not displayed.
    /* 动画效果 */
  transition: max-height 0.15s ease-out, opacity 0.15s ease-out !important;
    • Set a “transition animation (过渡动画)” for the max-height and opacity properties.
    • This line of code tells Firefox : “When you need to hide the toobar (that is, when you change from ‘displayed’ back to this’hidden’ state), don’t do it immediately”.
    • max-height 0.15s ease-out : within 0.15 seconds, smoothly fold uo the height (ease-out means the animation starts quickly and ends slowly).
    • opacity 0.15s ease-out: Meanwile, smoothly turning it transparent within 0.15 secconds.
    /* --- 鼠标悬停在顶部时，显示所有工具栏 --- */
#navigator-toolbox:hover #TabsToolbar,
#navigator-toolbox:hover #nav-bar,
#navigator-toolbox:hover #PersonalToolbar {
  /* ... */
}
    • #navigator-toolbox: This is the entire top area that includes all toobars.
    • #navigator-toolbox:hover when the mouse hovering (:hover) this entire top area.
    • When the mouse moves to the top area, select all the toolbars.
    /* 100px 足够高，能容纳所有工具栏 */
  max-height: 100px !important;
    • Force the “max-height” of the toolbar to be set to 100 pixels.
    • This will “release” the height limit. The element will expand from 0px to the actual height of its content (as long as it does not exceed 100px, 100px is just a large enough safety value).
    opacity: 1 !important;
    • Force the “opacity” to be set to 1 (100%) completely opaque (完全不透明)
    /* 动画效果 */
  transition: max-height 0.15s ease-in, opacity 0.15s ease-in !important;
    • Similarly, set transition animations for max-height and opacity.
    • max-height 0.15s ease-in smoothly expand the height from 0px to 100px within 0.15 seconds (ease-in means the animation starts slowly and ends quickly).

    Can it be turned on and off via a shortcut key?

    ==You can’t creat a “real-time (实时)” switch button or shortcut key for the code in use userChrome.css.

    • The userChrome.css file iss loaded when Firefox startsup. Once loaded, it will remain in effect until you restart Firefox next time. Browsers do not have built-in functions that allow you to dynamically open or close a certain piece of CSS code in this file while it is running.
    • Any extension that can achieve this (such as userChromeJS) is extremely complex and requres you to install a script loader and write Javascript, which goes far beyond modifying CSS.

    However, we can think it in a different way. We can modify the CSS code to make it utilize an existing feature of FIrefox as a switch. ==:has() The selector does not work correctly in the current firefox version 144.0. So, just take a look at the following ideas and it cannot achieve quick opening and closing and automatic hiding as expected.==

    /* --- 
  开关逻辑：
  仅当书签栏被用户设置为“隐藏”时 (collapsed="true")，
  才激活下面的所有自动隐藏规则。
--- */

/* 1. 默认隐藏 (仅在书签栏被隐藏时生效) */
#navigator-toolbox:has(#PersonalToolbar[collapsed="true"]) #TabsToolbar,
#navigator-toolbox:has(#PersonalToolbar[collapsed="true"]) #nav-bar,
#navigator-toolbox:has(#PersonalToolbar[collapsed="true"]) #PersonalToolbar {
  max-height: 0px !important;
  opacity: 0 !important;
  overflow: hidden !important;
  
  /* 隐藏动画 */
  transition: max-height 0.15s ease-out, opacity 0.15s ease-out !important;
}

/* 2. 悬停显示 (仅在书签栏被隐藏时生效) */
#navigator-toolbox:has(#PersonalToolbar[collapsed="true"]):hover #TabsToolbar,
#navigator-toolbox:has(#PersonalToolbar[collapsed="true]):hover #nav-bar,
#navigator-toolbox:has(#PersonalToolbar[collapsed="true"]):hover #PersonalToolbar {
  max-height: 100px !important; 
  opacity: 1 !important;

  /* 显示动画 */
  transition: max-height 0.15s ease-in, opacity 0.15s ease-in !important;
}

/* 3. (可选但推荐) 当点击地址栏时也显示 */
#navigator-toolbox:has(#PersonalToolbar[collapsed="true]):focus-within #TabsToolbar,
#navigator-toolbox:has(#PersonalToolbar[collapsed="true]):focus-within #nav-bar,
#navigator-toolbox:has(#PersonalToolbar[collapsed="true]):focus-within #PersonalToolbar {
  max-height: 100px !important; 
  opacity: 1 !important;
  transition: max-height 0.15s ease-in, opacity 0.15s ease-in !important;
}
    • #navigator-toolbox:has(#PersonalToolbar[collapsed="true"])
      • #navigator-toolbox means the whole toobar area.
      • :has(...) is a condition which means “when it contains…”
      • #PersonalToolbar[collapsed="true"] when an element with the id PersionalToolbar (bookmark bar) and its status is collasosed="true" (collapsed= hidden)
      • The subsequent {...} (automatic hiding rules)will only be applied when the user sets the bookmark bar to hidden. When you press Ctrl + Shift + B to display the bookmarks bar, the codition [collapsed="true"] is no loger met (满足), all auto-hide CSS will immediately become invalid, and tthe toobar will return to its original state.
